# Family Recognition Lab v1.0

Status: Research operating system  
Code name: R001  
Chief Scientist objective: Discover how a family can be understood better than any AI generator.  
Scope: Research platform, experiment database, evaluation framework, human review system, learning loop, and three-year research roadmap.

This document is not a product specification, prompt guide, image generation plan, database migration, feature backlog, deployment plan, or Growth Engine document.

The Family Recognition Lab exists to answer one question:

How can MyKinLegacy understand a family so well that the final collection makes them say, "This feels like us"?

## 1. Research Platform Design

The Family Recognition Lab is a research system for studying recognition, not a production system for generating artifacts.

Its purpose is to make every identity interpretation experiment observable, comparable, repeatable, and improvable.

### 1.1 Core Research Principle

The Lab does not optimize for beautiful images first.

It optimizes for family recognition:

- Does the interpreted identity feel specific?
- Does the meaning connect to the family story?
- Does the narrative feel emotionally true?
- Does the visual direction support recognition?
- Would a real human recipient keep it, gift it, or display it?

### 1.2 Research Unit

The atomic unit is a Recognition Experiment.

Each Recognition Experiment tests one research question or hypothesis by changing a controlled variable in the interpretation process.

Examples of variables:

- interview question order
- life moment presence
- grandparent memory depth
- value prioritization
- number of symbols
- narrative tone
- art direction specificity
- identity archetype
- symbol explanation style
- human review question format

### 1.3 Experiment Types

| Experiment type | Purpose | Example |
| --- | --- | --- |
| Interview Experiment | Test what input questions improve recognition | Ask about grandparents before animals |
| Interpretation Experiment | Test identity layer decisions | Compare value-first vs ancestry-first interpretation |
| Meaning Experiment | Test symbol-to-story mapping | Oak as strength vs oak as grandfather memory |
| Narrative Experiment | Test story logic | Gift narrative vs legacy narrative |
| Art Direction Experiment | Test visual translation constraints | Minimal symbol hierarchy vs dense crest |
| Collection Experiment | Test artifact sequence | Certificate first vs story first |
| Human Review Experiment | Test evaluation method | Open-ended review vs scored review |
| Comparative Experiment | Test MyKinLegacy against generic generation | Prompt-only output vs interpretation-first output |

### 1.4 Experiment Lifecycle

Every experiment must be:

1. Designed with a hypothesis.
2. Run against a defined input set.
3. Evaluated with a consistent recognition rubric.
4. Compared against a baseline or control.
5. Recorded in the Research Database.
6. Converted into a learning.
7. Mapped to next action.

### 1.5 Research Artifacts

Each experiment may produce:

- raw family input
- structured interview answers
- identity layer notes
- meaning graph
- narrative direction
- art direction
- generated collection sample
- human review notes
- evaluation scores
- interpretation learnings
- next iteration recommendation

### 1.6 Comparison Model

The Lab compares outputs at multiple levels:

| Comparison | What it proves |
| --- | --- |
| Prompt-only vs interpretation-first | Whether interpretation improves recognition |
| Short interview vs deep interview | Whether more input improves or overloads recognition |
| Values-first vs ancestry-first | Which input type better predicts recognition |
| Symbol-heavy vs narrative-heavy | Whether symbols or story carry identity more strongly |
| AI-only review vs human review | Whether machine confidence matches human recognition |
| Gift recipient vs purchaser review | Whether the artifact works for both giver and receiver |

### 1.7 Iteration Rule

An experiment is not complete when output is generated.

An experiment is complete only when the Lab records:

- what changed
- what improved
- what worsened
- what remained unknown
- what should be tested next

## 3. Research Workflow

The Lab follows a repeatable workflow:

Hypothesis -> Design -> Generate -> Evaluate -> Improve -> Repeat

### 3.1 Workflow Stages

| Stage | Purpose | Required output |
| --- | --- | --- |
| Hypothesis | Define what might improve recognition | Testable claim |
| Design | Choose input, control, variable, review method | Experiment plan |
| Generate | Produce interpretation and optional collection output | Research artifact |
| Evaluate | Score recognition using human and structured rubrics | Evaluation record |
| Improve | Decide what changed and why | Learning note |
| Repeat | Run the next iteration or retire the idea | Next action |

### 3.2 Research Rules

1. Change one primary variable per experiment.
2. Always keep a control or baseline when possible.
3. Separate image beauty from recognition quality.
4. Do not treat AI confidence as human recognition.
5. Always capture open-ended human notes.
6. Record failures; failed experiments are valuable.
7. Never operationalize a finding from one weak test.
8. Study gift-giver and recipient reactions separately.
9. Protect sensitive family contexts during review.
10. Prefer repeated small experiments over large vague studies.

### 3.3 Minimum Valid Experiment

An experiment is minimally valid when it has:

- a clear hypothesis
- one controlled variable
- at least one baseline
- at least three evaluation dimensions
- human notes
- a next action

### 3.4 Research Ethics

Family identity, grief, adoption, cultural heritage, and memory are emotionally sensitive. The Lab must avoid exploiting pain, fabricating ancestry, or pushing false authority.

Research must protect:

- privacy
- consent
- cultural sensitivity
- grief sensitivity
- family complexity
- non-official symbolic framing

## 7. Recognition Evaluation

Recognition Evaluation measures identity understanding, not image prettiness.

### 7.1 Evaluation Dimensions

Score each dimension from 1 to 5.

| Dimension | Definition | 1 means | 5 means |
| --- | --- | --- | --- |
| Identity Recognition | The family feels specifically understood | Generic | Clearly family-specific |
| Meaning Accuracy | Symbols match the intended meaning | Random symbols | Symbols fit deeply |
| Narrative Authenticity | Story direction feels emotionally true | Forced or invented | Natural and true |
| Family Recognition | "This feels like us" likelihood | Unlikely | Very likely |
| Gift Worthiness | Worth giving to recipient | Not giftable | Emotionally giftable |
| Print Worthiness | Worth printing or displaying | Disposable | Display-worthy |
| Keepsake Value | Worth saving long term | Temporary | Heirloom-like |
| Legacy Value | Helps preserve what matters | No continuity | Strong future meaning |
| Emotional Resonance | Tone fits the moment | Emotionally wrong | Emotionally precise |
| Symbol Coherence | Elements reinforce each other | Disconnected | Unified meaning |
| Cultural Care | Respectful heritage handling | Risky | Careful and safe |
| Privacy Respect | Sensitive details handled well | Exposes too much | Safe and appropriate |
| Trust Clarity | Claims and limits are clear | Confusing | Clearly trustworthy |
| Visual Translation Fit | Visuals reflect interpretation | Pretty but unrelated | Visually aligned |
| Completeness | Feels like a collection | Fragmented | Complete and coherent |

### 7.2 Aggregate Scores

| Score range | Level | Meaning |
| --- | --- | --- |
| 15-34 | Poor | Generic, unsafe, or not recognizable |
| 35-54 | Good | Usable but not deeply distinctive |
| 55-69 | Excellent | Strong recognition and collection fit |
| 70-75 | World Class | Deeply recognizable, giftable, and keepsake-worthy |

### 7.3 Non-Negotiable Failure Conditions

An output fails regardless of score if it:

- claims official heraldic authority
- invents genealogical proof
- copies protected logos or official emblems
- mishandles grief with manipulative language
- exposes private family details inappropriately
- uses culturally sensitive symbols irresponsibly
- makes AI the source of identity truth

## 8. Human Review Framework

The Lab must evaluate with real human judgment, not AI confidence alone.

### 8.1 Reviewer Types

| Reviewer type | What they reveal |
| --- | --- |
| Purchaser | Whether it solves the buying intent |
| Recipient | Whether the gift lands emotionally |
| Family member | Whether recognition is real |
| Neutral reviewer | Whether meaning is understandable |
| Sensitive-context reviewer | Whether grief, adoption, heritage, or family complexity is handled carefully |
| Internal expert reviewer | Whether framework standards are followed |

### 8.2 Core Review Questions

1. Does this feel like it belongs to this family?
2. What part feels most personal?
3. What part feels generic?

5. What feels emotionally wrong?

7. Which symbol feels unnecessary?
8. Does the story feel true to the input?

11. Would you keep this?
12. Would you print or display this?
13. Would the recipient understand why it was made?
14. Does the disclaimer feel clear without ruining the emotion?
15. Does it feel private and respectful?

17. What should be removed?

## 9. Learning Loop

The Learning Loop trains interpretation, not the AI model.

The goal is not to fine-tune an image model first. The goal is to improve how MyKinLegacy understands families before generation.

### 9.1 Learning Sources

- human review notes
- recognition scores
- failed experiments
- refund reasons
- support complaints
- recipient reactions
- open-ended "feels like us" explanations
- symbol confusion
- narrative mismatch
- privacy discomfort
- cultural sensitivity flags
- high-performing collection patterns

### 9.2 Learning Conversion

| Raw signal | Interpretation learning |
| --- | --- |
| "Too generic" | Need stronger identity archetype |
| "Pretty but not us" | Visual translation detached from meaning |
| "This part made me cry" | Emotional anchor identified |
| "Why this animal?" | Meaning graph too weak |
| "This feels fake historical" | Trust boundary failed |
| "This would mean a lot to my dad" | Gift intent aligned |
| "I would frame this" | Print worthiness signal |
| "This is too much" | Symbol overload |
| "This feels respectful" | Cultural or grief handling worked |

### 9.3 Monthly Research Review

Every month, the Lab should review:

- top recognition wins
- top recognition failures
- highest-scoring interpretation patterns
- lowest-scoring question paths
- confusing terms
- strongest gift moments
- strongest memory moments
- repeated cultural or trust risks
- experiments ready to operationalize
- experiments to retire

### 9.4 Operationalization Rule

A research learning becomes a standard only when it is:

- repeated across multiple scenarios
- supported by human review
- not limited to one AI model
- compatible with trust principles
- useful across at least one life moment cluster
