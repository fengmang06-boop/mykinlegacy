AI Image Generation Bridge v1

This bridge fixes the actual PNG bottleneck: the LRE prompt and PVE can now feed a real image provider for allowlisted orders. The deterministic renderer remains as the safe fallback.

Current Provider Readiness

OpenAI path: configurable for allowlisted LRE PNG artifacts.

Default production: unchanged unless provider env and order allowlist are set.

Fallback: deterministic renderer still produces delivery-safe PNGs.

Required Env

  • LRE_IMAGE_PROMPT_ORDER_ALLOWLIST=<order_number>
  • LRE_IMAGE_GENERATION_PROVIDER=openai
  • OPENAI_API_KEY=<secret>
  • OPENAI_IMAGE_MODEL=gpt-image-1
  • OPENAI_IMAGE_SIZE=1024x1024
  • OPENAI_IMAGE_QUALITY=high

New Execution Path

Order paid or artifact repair
Manifest-driven generation
PNG artifact branch
LRE Prompt Builder allowlist check
Prompt Verification Engine score must be 95+
Configured OpenAI image provider generates PNG
PNG metadata records provider, model, prompt source, PVE score, fallback flag
If provider is missing or fails, deterministic renderer is used and marked

Recorded Metadata

  • image_generation_bridge
  • image_provider
  • image_model
  • provider_request_id
  • prompt_source
  • pve_score / pve_passed
  • fallback_used / bridge_error

Conclusion

One allowlisted order can now generate real AI PNG artifacts when the OpenAI image provider is configured and PVE passes. Non-allowlisted orders remain untouched.
